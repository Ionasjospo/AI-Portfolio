Predict of Uruguay football match results

Introduction:
Based on the Kaggle dataset available in International Football Results from 1872 to 2017, a specific analysis was conducted to predict the outcomes of Uruguay's national team matches. This dataset includes three CSV files, but only the ‘results’ file was used. Given that the free version of RapidMiner limits visualization to the first 10,000 rows, the file was filtered to include only matches involving Uruguay. This selection was performed in Excel, transforming the original CSV into one containing only Uruguay matches.

Why did I choose this case study?
I chose this study because I am a fan of both football and computer science. This motivated me to combine both interests into a project that uses historical match data to generate predictions.

Objectives:
Using the data provided by the dataset, the goal is to predict whether Uruguay will win, lose, or draw in different matches using data analysis techniques and classification models.

Using the Generate Attributes operator, we created a new column called Uruguay_results. This column indicates whether Uruguay won, drew, or lost the match. The logic within the operator is as follows:

if(home_team == "Uruguay" && home_score > away_score, "Won",
   if(home_team == "Uruguay" && home_score < away_score, "Lost",
      if(away_team == "Uruguay" && away_score > home_score, "Won",
         if(away_team == "Uruguay" && away_score < home_score, "Lost",
            "Draw"))))

The distribution is: It has a categorical distribution of Uruguay's results. The distribution shows that the team has won most matches, followed by losses and, to a lesser extent, draws.

Naive Bayes Model Application

A process was created in RapidMiner applying the Naive Bayes model to predict Uruguay's match outcomes.

The model's performance shows an overall accuracy of 49.82%. The "Won" class has the highest precision (60.18%), while the "Draw" class is the lowest (29.27%). This suggests that the model identifies wins better, but struggles with predicting draws. The low success rate indicates a need for data optimization or testing other models to improve prediction accuracy.

Optimization of Accuracy
  • Normalizing and standardizing the data did not improve accuracy.
  • The Select Attributes operator had issues, so other attribute selection techniques were used.
Feature Selection with Forward Selection

The process was configured to use the Forward Selection operator, identifying the most relevant attributes: home_team, home_score, away_score, and neutral. This resulted in an overall accuracy improvement, increasing to 65.61%.

Comparison of Confusion Matrices:
  • Overall accuracy improvement: Increased from 49.82% to 65.61%, indicating better model performance by removing irrelevant attributes.
  • Class precision: The precision of the "Won" class improved from 60.18% to 70.63%, and the "Lost" class increased from 55.56% to 73.68%. The "Draw" class also improved, going from 29.27% to 50.60%.
Other Feature Selection Techniques
  • The Backward Elimination technique improved accuracy, but was inferior to Forward Selection.
  • Optimize Selection matched Forward Selection's accuracy but did not exceed it.

Conclusion

Decision Tree vs. Naive Bayes for Predicting Uruguay's Match Outcomes

After incorporating a Decision Tree model alongside the Naive Bayes model, the results showed that the Decision Tree yielded a lower accuracy compared to the Naive Bayes approach. This suggests that, for this particular dataset, Naive Bayes is better at capturing the underlying patterns in the match data of Uruguay.

Possible Reasons:

  • Feature Independence: Naive Bayes assumes that all features are independent, which, despite being a strong assumption, can perform well when the assumption holds reasonably true. It might be more suitable for this dataset if the input features do not have strong interactions.
  • Decision Boundaries: Decision Trees tend to create hard splits and might struggle when the decision boundaries between winning, losing, and drawing are not well-defined in the feature space. This can lead to overfitting in the training set, causing a decrease in performance on unseen data.
  • Data Distribution: The nature of football match results (win, draw, lose) may be better modeled by the probabilistic approach of Naive Bayes, especially given the skewed distributions in some features like scores.

Recommendation:

While Decision Trees offer interpretability by showing how different features influence outcomes, Naive Bayes seems more effective for this specific problem in terms of accuracy. To further improve model performance, it could be beneficial to try ensemble methods like Random Forests, which combine multiple decision trees and often provide better predictive power.

Additionally, exploring other classification models such as Support Vector Machines (SVM) or even deep learning techniques could yield better results in predicting match outcomes.
