Introduction to Machine Learning

In Unit 1, we cover essential techniques and tools in high demand in the industry, focusing on the Data Science and Machine Learning process. We explore the CRISP-DM methodology for development, which guides the workflow from understanding data to deploying models. Different types of algorithms and their applications are introduced, along with the importance of building machine learning portfolios. The unit also addresses desirable characteristics of datasets used for training ML models and types of problems they can solve, such as those from the UCI repository. We review modern tools like Weka and RapidMiner, and cover basic statistical concepts and spreadsheet usage for data handling.

CRISP-DM

What is CRISP-DM?
Cross-Industry Standard Process for Data Mining is a widely used methodology for data mining and data science projects, developed in 1999. It consists of six key phases:
1. Business Understanding: Define project goals from a business perspective, assess resources, and set both business and data mining objectives. 2. Data Understanding: Identify and explore data, verify its quality, and document its properties.
3. Data Preparation: Clean and format the data, select relevant features, and integrate datasets for modeling.
4. Modeling: Select and apply algorithms, build models, and assess their performance.
5. Evaluation: Ensure the models align with business goals and evaluate the process before deciding next steps.
6. Deployment: Implement the model, create reports, and plan for long-term monitoring and maintenance.
CRISP-DM is adaptable and can be combined with agile project management for optimal results.

UCI Machine Learning Repository

What is UCI?
The UCI dataset is a data repository maintained and made available by the University of California, Irvine that is widely used for machine learning and data mining research. You can browse datasets at UCI archive.
